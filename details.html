MusicStore

Tech used

This is an e-commerce app that can be used to sell any category of products. The only thing that is needed is uploading the product images and information for the products, and of course getting users.

This app has a number of nice features like pagination for the products on the home page and for the admin pages. The user also has the option to view their orders and to edit their name, email address, and password.

I deployed the live version on Render or take a look at my code on GitHub.

Why I built this app

I feel that an e-commerce app is the pinnacle of web development that is not enterprise-level. I hope to be able to make improvements to this app as I continue to review the code.

Challenges encountered

This project had many moving parts which was overwhelming at times. The part that I found most challenging was using React Redux.

What did I learn?

This was the second time I built an app using the MVC architecture and I would like to get a better fell for that approach. I also learned to create collections in Postman which was extremely helpful.

Is the app complete?

Almost. I would like to give the user the option to edit and/or delete their product reviews. I also have style changes I would like to make on the admin pages. I would also like to add a Stripe payment option.

YelpRetreat

Tech used

This project is a Yelp-like review site for various types of retreats. The retreats can be found via the cluster map, or by retreat type.

The best feature of the app is the ability for users to create accounts and have their passwords, usernames, and emails stored in Mongo. Then a user can create reviews for existing retreats or create their own retreat listing.

I deployed the live version on Render or take a look at my code on GitHub.

Why I built this app

YelpRetreat is a variation of a full-stack app that I built. I wanted to change not only the styles but also the complexity by creating different types of retreats.

Challenges encountered

I found the documentation for Mongoose to be very confusing. It took a long time to find the methods and options that I needed.

What did I learn?

Learning the basic Express server setup became second nature. It was not so intuitive to change the project structure to an MVC pattern. Also, working with Mongo Shell and Mongoose was very helpful in introducing me to back-end technology.

Is the app complete?

There are features and style changes that I would like to make. One thing I would like to add is to prevent a retreat owner from being able to add a review to their own listing. It would also be nice to have an average overall rating for each retreat.

Writing Assistant

Tech used

I built this app for my aunt who has cerebral palsy. My hope was that she would be able to select words and phrases for better emails to friends and family.

If you know someone with a disability who could use this tool then send them to the live version and have someone get in touch with me. Also, take a look at the code on GitHub.

Why I built this app

It is for anyone with reduced fine motor control who finds it easier to click with an assistive mouse than using a keyboard. The hope is that they will be able to type more text in less time.

How I built this app

I added the most common words to an alphabetical mega menu, added the functionality for the keyboard keys, added the textarea with clear/copy buttons, and lastly, I added the proper nouns section behind an accordion button.

What did I learn?

It is absolutely essential to have a clear plan outlined of what you need to accomplish, and I learned that the hard way. This was my first project and the details were not clear in my mind.

Is the app complete?

No, I need to use the File API so users can upload a file of proper nouns and sample text so that they can have a custom-built page. I would also like to create keyboards for other languages.

Guitar Chord Namer

Tech used

This app returns the name of a guitar chord, the chord intervals, the chord tendencies, whether or not it equals another chord by a different name and the scales that build the chord.

You won't find a more accurate and complete guitar chord namer app! If you or someone you know plays guitar then try out the app on my guitar website or check out the code on GitHub.

Why did you build this app?

I started this project after viewing SVG Guitar Chord repos on GitHub and noticed that their names were often wrong. I also was not able to find any quality websites on the subject.

How I built this app

I had to create a lot of variables for my form elements, output fields, and music note arrays. I also had to write all the for loops and conditions within a single function.

What did I learn?

It was a challenge building the majority of the functionality inside a single function that is called from the form event listener.

Also. the use of the JS every method was the main "helper" method for getting the functionality I needed.

Is the app complete?

This app is complete for its purpose but it can be much better. I would like to have an option for ukeleles, banjos & mandolins.

I also intend to load the users' tuning and sharp/flat selection into local storage.

TMDB Flixx App

Tech used

This app uses The Movie Database API to show cards for the most popular movies and TV shows. The cards have links to the details for each show or movie. On the details page, there are also links to YouTube videos and to a Cast and Crew page for the show/movie.

The cast and crew page has cards for the cast members with links to a personal page for each cast member. The crew cards are the same but I split the cards up by department.

You can take a look at the live version on Netlify and the code on GitHub.

Why this app for my portfolio

The design and styles were from a Brad Traversy project. I prefer to design my own projects, but the design is nice. I chose to add this project to my portfolio because I added additional API calls and details that were not part of the course project.

Challenges encountered

It took me some time to figure out how to get the movie/show title onto the cast and crew page. I also had to create different cards for members with or without a photo. Finally, sectioning the crew by department took a good amount of time and code.

What did I learn?

Working with an API and using the results from fetching data to make another API call was insightful. I also became very comfortable properly manipulating the dom with methods such as createElement, createTextNode, and setAttribute.

Is the app complete?

I would like to use Netlify functions so that I can hide my API key but I am unsure how to use export.handler with all the files that rely on my API key, or the results from fetching the data.

Tarp Configurations Calculator

Tech used

This is a personal project that aligns with my interests in bushcraft and wilderness survival. It calculates the height of the rope(s) used to raise your tarp for 33 different tarp configurations.

It also notes whether the user is able to 1) sit under the tarp, and 2) sit in a chair under the tarp.

You can try out the app on Netlify or take a look at the code on GitHub.

Why I built this app?

I wanted to do something unique so I converted my spreadsheet equations for tarp configuration into a React/JS application.

This app would be useful for campers, backpackers, and bushcrafters.

Challenges encountered

I had to do some research for my Hamburger menu and PNG files that were not outputting due to their small size and default Webpack settings.

I also had to redesign the overall file structure due to re-renders of components.

What did I learn?

The ease of Tailwind and the basics of React such as react-router-dom, use hooks (useState, useContext, useEffect, useRef, useNavigate, and useLocation), and conditional JSX.

Final thoughts

Using useContext was better than useState but I would like to incorporate useReducer for my state values. I would also prefer to create my own webpack.config file for greater control.
